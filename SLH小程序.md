## 二、首页（Home Tab）

### 2.1 页面定位

首页是用户进入小程序的第一触点，承担品牌展示、酒店发现与预订发起三重功能。整体视觉基调以暖米色 + 深棕色为主，与 SLH（Small Luxury Hotels of the World）品牌调性保持一致。

### 2.2 功能模块说明

#### 2.2.1 顶部 Banner — 品牌展示区

⚠️ **Tab一级页面顶部导航栏说明**：首页为一级Tab页面，顶部仅展示品牌Logo，**无返回按钮**。返回按钮仅出现在二级页面。

| 元素 | 说明 |
|------|------|
| 品牌 Logo | 页面顶部居中展示"SLH" |
| Hero 图 | 全幅展示当前主推内容（酒店图 / 优惠活动图），支持多张轮播 |
| Swipe 指示器 | 底部圆点表明可左右滑动，支持多张内容轮播 |

**内容类型**：Banner 展示内容分为两类：
- **酒店图**：展示主推酒店大图（如：西安尤辰酒店 YOUNCE HOTEL XI'AN）
- **优惠活动图**：展示当前促销活动主题图

**交互说明**：点击 Banner 区域 → 根据运营配置的链接跳转（可跳转至酒店详情页 / 优惠活动详情页 / 优惠活动列表页等，由 CMS 配置）。

**开发说明**：Banner 内容支持 CMS 配置，可按酒店/季节/活动更新；轮播间隔建议 4s，支持手动滑动。

---

#### 2.2.2 搜索入口区（Quick Search）

**功能描述**：用户可在首页直接发起酒店搜索，无需跳转至"酒店"Tab，降低预订路径长度。

**交互流程**：

```
点击"酒店、国家、城市"输入框 → 跳转【搜索页（默认态）】
点击"选择日期"区域 → 跳转【选择日期页】→ 确认 → 返回首页，搜索入口更新日期
点击"人数"区域 → 跳转【旅客人数页】→ 确认 → 返回首页，搜索入口更新人数
点击"搜索全球精品酒店"按钮 → 跳转【搜索页（默认态）】
```

**字段说明**：

| 字段 | 类型 | 必填 | 默认值 |
|------|------|------|--------|
| 目的地 | 文本输入（联想搜索） | 否 | 空，Placeholder："酒店、国家/城市" |
| 入住日期 | 日期选择 | 否 | 空，Placeholder："选择日期" |
| 离店日期 | 日期选择 | 否 | 空，联动入住日期 |
| 旅客人数 | 数字选择 | 否 | 成人 2，儿童 0 |

**边界条件**：
- 目的地为空时点击搜索：弹出提示"请选择目的地"
- 入住日期 ≥ 离店日期：不允许确认，提示"离店日期须晚于入住日期"
- 成人人数最少 1 人，上限 **10** 人
- 儿童人数最少 0 人，上限 **6** 人
- 日期/人数修改后确认：数据回填至首页搜索入口，无需清空其他字段

---

#### 2.2.3 热门酒店（Top Hotels）

**功能描述**：以横向滚动卡片形式展示精选酒店，引导用户发现与探索。

**卡片信息结构**：
- 酒店图片（4:3 比例，全幅）
- 酒店中文名（主标题，加粗）
- 酒店英文名（副标题，小字）
- 所在城市 + 国家

**交互**：
- 左右滑动浏览更多酒店
- 点击卡片 → 跳转酒店详情页

**内容来源**：由运营后台配置，建议支持权重排序与酒店分组标签（如"热门""新上线"）。

---

#### 2.2.4 优惠活动（Promotions）

**功能描述**：以横向滚动 Banner 卡片形式展示当前促销活动，点击进入活动详情页。

**卡片信息结构**：
- 活动主题图（深色风格，高对比度设计）
- 活动名称（如：SAFE DAN 0折）
- 活动标签文字

**交互**：
- 左右滑动浏览更多活动
- 点击卡片 → 跳转【优惠活动详情页】
- 点击"查看全部"→ 跳转【优惠活动列表页】

---

#### 2.2.5 微信商城（WeChat Store）

**功能描述**：以横向滑动卡片形式展示多个微信商城入口，与 SLH 周边商品/礼品卡等电商业务打通。

**交互**：
- 左右滑动浏览多个商城卡片
- 点击卡片 → 通过微信小程序跳转能力，打开对应外部商城小程序的**指定产品页**（使用 `wx.navigateToMiniProgram`，需配置 appId 白名单）；跳转时**无需登录态**

---

#### 2.2.6 底部弹窗 — 关注公众号

**功能描述**：小程序首次进入时，底部弹出关注公众号引导，提升品牌私域触达能力。

**交互**：

| 操作 | 行为 |
|------|------|
| 点击"×"关闭 | 弹窗关闭，**本次会话不再出现**（通过本地缓存记录关闭状态），下次重新打开小程序时再次展示 |
| 点击"去关注" | 跳转至同主体公众号的**文章页**（非公众号主页，需确认跳转链路） |

**技术说明**：
- 关闭状态建议存储在小程序本地 Storage（`is_subscribe_popup_closed`）
- **每次冷启动均重置此字段**（不持久化），即每次冷启动都会重新展示弹窗，无最大展示次数限制
- "去关注"跳转使用微信开放能力，需提供公众号文章 link

## 三、酒店（Hotel Tab）

### 3.1 页面定位

小程序底部Tab第二个入口，与首页Tab平行。承担主题酒店发现、热门酒店推荐、目的地探索、酒店发现等功能入口。

### 3.2 功能模块说明

#### 3.2.1 顶部导航栏

⚠️ **Tab一级页面顶部导航栏说明**：酒店Tab为一级Tab页面，顶部仅展示页面标题"酒店"，**无返回按钮**。返回按钮仅出现在二级页面。

#### 3.2.2 主题酒店入口区

**功能描述**：展示主题酒店分类入口，用户可浏览所有主题分类，也可快速进入某个主题下的酒店列表。

**区域标题栏**：标题"主题酒店推荐" + 右侧"查看全部"文字入口（点击 → 进入【全部主题汇总页】（3.5））。

**卡片结构**：2列网格布局，每个卡片包含：

| 字段 | 说明 |
|------|------|
| 主题封面图 | 3:4 比例圆角图片 |
| 主题中文名 | 主标题（如"野奢酒店"、"古堡酒店"、"米其林酒店"、"极光酒店"） |
| 主题英文名 | 副标题，大写字母（如"WILD LUXURY"、"CASTLES & MANORS"） |
| 更多入口 | "更多 »" 文字链接，点击 → 进入【主题酒店列表页】 |

**布局规则**：2列等宽网格，间距均匀。

**交互说明**：
- 点击卡片任意区域 → 进入【主题详情页】
- 点击"更多"文字 → 进入【主题酒店列表页】

#### 3.2.3 热门酒店

**功能描述**：以横向滚动卡片形式展示精选热门酒店。

**卡片信息结构**：
- 酒店图片（4:3 比例，全幅）
- 酒店中文名（主标题，加粗）
- 酒店英文名（副标题，小字）
- 所在城市 + 国家

**交互**：
- 左右滑动浏览更多酒店
- 点击卡片 → 跳转酒店详情页

#### 3.2.4 热门目的地

**功能描述**：展示两类热门目的地入口：大中华区 / 海外。

**布局**：两个目的地类别并列展示，每个类别可展开查看具体目的地城市列表。

**交互说明**：
- 点击目的地类别 → 展开/收起该类别下的城市列表（**不跳转页面**，仅在当前区域下方展开城市列表）
- 点击具体城市 → 跳转【酒店列表页】，以该城市为筛选条件

#### 3.2.5 最新成员酒店

**功能描述**：以横向滚动卡片形式展示最新加入SLH的酒店。

**卡片信息结构**：
- 酒店图片（4:3 比例，全幅）
- 酒店中文名 + 英文名
- 所在城市 + 国家

**交互**：
- 左右滑动浏览更多酒店
- 点击卡片 → 跳转酒店详情页

#### 3.2.6 底部导航栏

四个Tab：首页 / 酒店（高亮） / 品牌 / 行程

### 3.3 主题详情页

**触发路径**：酒店Tab → 点击主题卡片

**页面结构**：

| 模块 | 说明 |
|------|------|
| 顶部导航栏 | 返回按钮 + 标题（主题中文名） |
| Hero 图区 | 全宽主题大图 + 主题中文名 + 主题英文名叠加展示 |
| 主题介绍 | 主题描述文案 |

## 九、行程（Itinerary Tab）

### 9.1 页面定位

小程序底部Tab第四个入口。承担订单查询功能，**无订单列表页**，仅支持通过以下条件查询订单：
- **确认号（Confirmation Number）** 或 **行程号（Itinerary Number）** + **姓氏（Last Name）**

**搜索结果页展示规则**：
- 通过**行程号**查询：若该行程包含多个房间，搜索结果页会展示**多个订单入口**（一个订单对应一个房间、一个确认号）
- 通过**确认号**查询：搜索结果页仅展示**该房间对应的单个订单入口**

### 9.2 功能模块说明

#### 9.2.1 顶部导航栏

⚠️ **Tab一级页面**：顶部仅展示"SLH"品牌Logo，**无返回按钮**。

#### 9.2.2 订单搜索功能

**输入字段**：两个独立输入框，均为**必填**：
1. **确认号（Confirmation Number）** 或 **行程号（Itinerary Number）**
2. **姓氏（Last Name）**

**交互**：两个字段均填写后 → 触发搜索 → 跳转搜索结果页

#### 9.2.3 联系我们入口

**位置**：行程Tab搜索输入框下方的提示文案中。

**交互**：点击 → 跳转联系我们页面

---

### 9.3 搜索结果页

**触发路径**：行程Tab → 填写确认号/行程号 + 姓氏 → 搜索

**⚠️ 二级页面**，顶部有返回按钮。

> **三级页面结构说明**（行程查询完整流程）：
> - **第一级**：行程Tab（9.2）— 搜索输入框
> - **第二级**：搜索结果页（9.3）— 查询结果梗概 + 订单细节入口
> - **第三级**：订单详情页（9.4）— 订单更多细节

**搜索结果卡片**：酒店名称 + 已预定房间信息 + 每个房间的订单详情入口

**交互**：点击房间订单入口 → 跳转该房间对应的订单详情页（9.4）

### 9.4 订单详情页

**触发路径**：搜索结果页 → 点击具体订单

**⚠️ 二级页面**，顶部有返回按钮。

**页面结构**：

| 模块 | 说明 |
|------|------|
| 顶部导航栏 | 返回按钮 + "SLH"品牌Logo |
| 行程号区域 | 行程号（如"22402B6917382"） |
| 酒店信息区 | 酒店中文名 + 英文名 |
| 订单状态标签 | 状态标签 + 指示点 |
| 房间列表区 | 多个房间子卡片 |
| 联系我们区域 | "CONTACT US / 联系我们"入口 |
| 底部操作栏 | **固定悬浮在页面底部，根据订单状态显示不同按钮** |

**房间子卡片**：房间标签 + 确认号 + 房间名称 + 入住日期 + 入住人数 + "VIEW DETAILS / 查看详情"入口（点击 → 跳转该房间订单更多细节页，即第三级页面，展示完整订单细节）

**底部操作栏（固定悬浮在页面底部）**：

| 订单状态 | 按钮 |
|----------|------|
| 已确认 | 订单可修改时显示"修改订单" / 订单可取消时显示"取消订单" / "返回酒店详情" |
| 已取消 | 仅"返回酒店详情" |

⚠️ **修改按钮**：仅在订单可修改时显示；**取消按钮**：仅在订单可取消时显示；两个按钮相互独立，各自按条件决定是否展示。

**交互说明**：

- **修改订单**：点击 → 跳转该订单关联的【酒店详情页】，重走预订流程
- **取消订单**：点击 → 弹出确认取消订单弹窗，弹窗内展示取消及退改政策提示（**内容由接口返回，格式固定，关键信息以接口数据替换**），询问用户是否确认取消；确认后订单状态变更为"已取消"
- **返回酒店详情**：点击 → 返回该订单关联的【酒店详情页】（**携带预订时的日期和人数参数**）
